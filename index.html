What is accessibility?

Accessibility is making your website usable by as many people, in as many settings, with as many varying devices as possible.

We can be sure we are at the top of the accessibility game by following the guidelines instituted by W3C, the World Wide Web Consortium, in WCAG.

WCAG stands for Web Content Accessibility Guidelines...

These guidelines make sure our technologies are Perceivable, Operable, Understandable, and Robust - for as many users as possible.

We know accessibility is important for those of us with physical, auditory, visual, or other disabilities.

What you may not know is how following accessibility guidelines is crucial to make sure all users are able to consume your information.

In what kinds of situations do people end up using assistive technologies?

Most commonly, accessibility refers to making the web accessible to individuals with disabilities. These can be auditory, cognitive, neurological, physical, speech, or visual disabilities.

However. Web accessibility also assists able-bodied, neurotypical individuals in certain situations such as...

  • People using mobile phones, smart TVs, or other devices with small screens or unusual input modes
  • People with changing abilities due to aging
  • People with temporary ability hindrances such as medication induced impairment, broken limbs, missing glasses or hearing aids
  • Situational limitations like bright sunlight or a quiet library with no headphones...

How are assistive technologies using our websites?

Screen readers allow people to navigate websites in a variety of ways.

How they get around:

  • “Focusable items” like links (href), buttons, input items, textarea items, objects, select items
    • Focusable items include links (href) and buttons by default, but also include any item that includes a “tabindex” attribute
  • Characters, words, objects, groups
  • Next heading, next list, next list item, next media, next list item, next table, etc

Perhaps the most important question, though, is how can we make sure our websites are fully accessible?

HTML Accessibility:

Make sure you are using HTML elements for their intended purpose. Replace 'div' elements with something more specific - perhaps 'nav', 'aside', 'article', 'footer', 'header', etc...

Assistive technologies use these elements to streamline a user's navigation process. Using a 'nav' element allows the user to skip over the navigation menu, rather than needing to repeat the same lines over and over.
